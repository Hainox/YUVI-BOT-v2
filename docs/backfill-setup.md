# Настройка и запуск backfill (загрузка старой истории чата)

## Что такое backfill и зачем он нужен

Обычный Telegram Bot API (то, через что работает сам бот) не отдаёт историю сообщений, отправленных ДО того, как бота добавили в группу. Бот видит только новые сообщения, начиная с момента добавления.

Backfill — это отдельный разовый процесс, который заходит в чат не от имени бота, а от имени ТВОЕГО личного Telegram-аккаунта (как обычное MTProto-приложение вроде официального клиента Telegram) и скачивает всю доступную историю сообщений чата, а затем сохраняет её в базу данных бота — так же, как если бы бот собирал эти сообщения сам, в реальном времени.

## 1. Где взять TG_API_ID и TG_API_HASH

**Важно понимать:** `TG_API_ID` и `TG_API_HASH` — это креды (учётные данные) ТВОЕГО ЛИЧНОГО Telegram-аккаунта, а не бота. Это НЕ то же самое, что `BOT_TOKEN` из `docs/botfather-setup.md`. Не перепутай:
- `BOT_TOKEN` — выдаёт BotFather для бота.
- `TG_API_ID` / `TG_API_HASH` — выдаёт сам Telegram для твоего личного аккаунта, чтобы приложения могли подключаться к Telegram от твоего имени (backfill использует именно твой личный аккаунт, потому что только у обычных аккаунтов есть доступ к полной истории чата через этот механизм).

Шаги получения:

1. Открой в браузере https://my.telegram.org и войди под своим личным номером телефона (тем, которым ты обычно пользуешься в Telegram).
   - **Ожидаемый результат:** Telegram прислал код подтверждения в приложение Telegram — введи его на сайте.
2. После входа перейди в раздел **API development tools**.
3. Заполни короткую форму (App title и Short name можно указать любые, например `yuvi-backfill`), Platform — `Desktop`.
4. Нажми **Create application**.
   - **Ожидаемый результат:** на странице появились два значения: `App api_id` (число) и `App api_hash` (длинная строка из букв и цифр).

**Эти два значения — секретные, как пароль.** Их нельзя публиковать, отправлять в чат, коммитить в git — только вставлять в `.env`.

## 2. Как заполнить .env

Открой файл `.env` в корне проекта (если его ещё нет — скопируй `.env.example` в `.env`) и заполни:

```
TG_API_ID=12345678
TG_API_HASH=abcdef1234567890abcdef1234567890
```

- `TG_API_ID` — вставь число из шага 1.4 (без кавычек).
- `TG_API_HASH` — вставь строку из шага 1.4 (без кавычек).

Файл `.env` не попадает в git (он в `.gitignore`) — это сделано специально, чтобы секреты не утекли в историю репозитория. Никогда не убирай `.env` из `.gitignore` и не добавляй его в коммит вручную.

Без `TG_API_ID`/`TG_API_HASH` бот запускается и работает нормально — просто команда `/backfill` не сможет запустить загрузку истории и покажет понятную ошибку со ссылкой на my.telegram.org.

## 3. Как запустить backfill

Есть два равноценных способа.

### Способ А: команда в чате (рекомендуется)

1. Убедись, что бот запущен (`docker compose up -d` или локальный запуск) и добавлен в группу.
2. В группе, от имени любого администратора чата (не обязательно владельца бота — см. ниже), отправь команду `/backfill`.
   - **Ожидаемый результат:** бот сразу отвечает в группе, что backfill запущен и история подгружается в фоне.
3. При первом запуске потребуется подтвердить вход в личный аккаунт: в консоли/логах бота появится запрос кода подтверждения — код придёт в приложение Telegram на твой телефон, введи его туда, куда просит консоль.
   - **Ожидаемый результат:** после ввода кода создаётся файл сессии, и в дальнейшем повторный ввод кода не потребуется (пока файл сессии не удалён).
4. Дождись завершения (для большой истории может занять от нескольких минут до пары часов — зависит от объёма чата).

**Кто может запускать `/backfill`:** любой ТЕКУЩИЙ администратор чата — бот на каждый вызов команды заново проверяет права через Telegram (не по старому списку, не по владельцу бота). Обычный участник чата без прав администратора получит отказ.

### Способ Б: отдельный скрипт (для ручного/разового запуска без чата)

```bash
python scripts/backfill_history.py
```

Скрипт использует те же `TG_API_ID`/`TG_API_HASH`/`CHAT_ID` из `.env`. Полезно, если нужно запустить загрузку истории до того, как бот вообще начал работать в чате, или для отладки.

## 4. Важное предупреждение про медиа из backfill

Фото, видео, стикеры и другие файлы, загруженные через backfill, помечаются в базе данных как `source=backfill` — это архивная метка, отличающая их от файлов, собранных ботом вживую.

**Почему это важно:** файл (`file_id`), полученный через личный аккаунт при backfill, физически не может быть повторно скачан через Bot API бота — это два разных типа доступа к Telegram, и file_id одного не работает в другом. Для таких файлов сохраняется только `file_unique_id` — стабильный идентификатор, полезный для статистики и защиты от дублей, но НЕ для повторной отправки файла через бота.

На практике это означает: backfill корректно восстановит статистику (сколько сообщений, кто когда писал, какие эмодзи использовались) по всей старой истории, но старые фото/видео из backfill нельзя будет переслать через бота повторно — только те, что бот соберёт сам, начиная с момента добавления в группу.

## 5. Backfill безопасно запускать повторно

Если запустить `/backfill` ещё раз (например, после того как в чате появилось больше сообщений, или просто по ошибке дважды подряд) — дублей в базе данных не будет. Уже загруженные сообщения не задваиваются и не портят статистику: повторный прогон просто ничего не найдёт нового для уже обработанных сообщений.
