# Настройка BotFather для Yuvi Bot v2

## Обязательно

1. Создай бота через `/newbot`.
   - **Ожидаемый результат:** BotFather прислал токен вида `123456:AAExample...`.
2. Скопируй токен и вставь в `.env` (`BOT_TOKEN`).
   - **Ожидаемый результат:** в файле `.env` строка `BOT_TOKEN=123456:AAExample...` (без кавычек).
3. Добавь бота в нужную группу.
   - **Ожидаемый результат:** бот появился в списке участников группы.
4. Выдай боту права администратора группы.
   - Открой группу → название группы → `Administrators` → `Add Admin` → выбери бота.
   - **Ожидаемый результат:** бот отображается в списке администраторов группы.
   - **Почему это обязательно, а не просто рекомендация:** см. раздел ниже про реакции — без прав администратора часть данных (реакции) не будет собираться вообще, даже если всё остальное настроено правильно.
5. В BotFather отключи privacy mode:
   - `/mybots` → выбери бота
   - `Bot Settings` → `Group Privacy`
   - **Turn off**
   - **Ожидаемый результат:** BotFather прислал сообщение `Privacy mode is disabled for <имя бота>`.

Без отключения privacy бот не увидит все сообщения, и статистика будет неполной.

## Отдельное требование: права администратора нужны для реакций (message_reaction)

Это отдельное, обязательное требование Telegram Bot API — оно НЕ связано с privacy mode из пункта 5 выше и не решается им.

**Если бот НЕ администратор группы:**
- Реакции (эмодзи-реакции на сообщения) вообще не приходят боту — Telegram Bot API просто не отправляет боту события `message_reaction`, если бот не в списке администраторов группы.
- Все остальные данные (сообщения, правки, стикеры, медиа) при этом продолжают собираться нормально — проблема касается только реакций.
- Таблица `reactions` в базе данных при этом останется пустой, даже если участники чата активно ставят реакции на сообщения — это не баг бота, а прямое ограничение Telegram.

**Как проверить, что права настроены верно:**

1. Открой группу, где работает бот.
2. Найди любое недавнее сообщение (можно своё) и поставь на него любую эмодзи-реакцию (например 👍).
3. Подожди несколько секунд.
4. Проверь таблицу `reactions` в базе данных (см. пример SQL-запроса в `docs/how-to-verify-phase1.md`, раздел «Реакции»).
   - **Ожидаемый результат:** появилась новая строка в таблице `reactions`, соответствующая только что поставленной реакции.
   - **Если строка НЕ появилась:** вернись к шагу 4 из раздела «Обязательно» выше и убедись, что бот действительно есть в списке администраторов группы (а не просто участник).

## Отдельное требование: право «закреплять сообщения» нужно для входа в Mini App (D-01/D-02)

Это отдельное, обязательное требование для казино-хаба (Mini App) — оно НЕ связано ни с privacy mode (пункт 5 «Обязательно»), ни с правом на реакции выше.

При старте бот автоматически публикует и закрепляет в группе сообщение с кнопкой «🎰 Открыть казино» (единственная точка входа в Mini App из группового чата — обычная WebApp-кнопка в группах не работает). Если у бота нет права «Закреплять сообщения», бот не сможет закрепить это сообщение (само сообщение при этом отправится, просто останется незакреплённым).

**Как проверить, что право выдано верно:**

1. Открой группу → название группы → `Administrators` → найди бота в списке.
2. Убедись, что у бота включено право **«Pin Messages»** (в интерфейсе может называться «Закреплять сообщения»).
   - Это отдельная галочка в списке прав администратора — общего права «Administrator» недостаточно, если конкретно эта галочка снята.
3. Перезапусти бота (или дождись следующего старта контейнера `bot`).
4. Открой группу и убедись, что сообщение с кнопкой «🎰 Открыть казино» появилось И закреплено (значок 📌 рядом с сообщением/в шапке чата).
   - **Ожидаемый результат:** сообщение видно в чате и отмечено как закреплённое.
   - **Если сообщение отправилось, но НЕ закрепилось:** вернись к шагу 2 и убедись, что галочка «Pin Messages» у бота действительно включена.
